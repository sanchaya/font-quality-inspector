What gets checked
🔡
Unicode Coverage
All 88 codepoints in U+0C80–U+0CFF including 12 revival/archaic characters. Required vs optional tracked separately.
⚙️
OpenType Features
All 21 GSUB/GPOS shaping features — nukt, akhn, blwf, vatu, cjct, pres, abvs, blws, psts, haln, mark and more.
📐
GSUB Rules
Per-feature lookup tables with rule counts, substitution types, and feature→lookup index mapping.
🎯
GPOS & Mark Anchors
Mark-to-Base (type 4) and Mark-to-Mark (type 6) lookups with anchor count. Critical for vowel sign attachment.
📋
Metadata
Name table fields, OS/2 weight/width class, Unicode range bits, typo/win metrics, italic angle.
📊
Quality Score
100-point score across coverage (40), features (30), scripts (10), metadata (10), and revival support (10). Grades A–F.
🔬
Cross-font Comparison
Compare any two fonts side by side. Generates a fix plan with missing GSUB rules, glyph gaps, and sample rules from the reference font as a blueprint.
📁
Glyph Inventory
Full glyph list with advance width statistics, empty glyph detection, and outlier flagging for Kannada glyphs.

Node.js CLI

For batch processing, CI pipelines, or local workflows. Saves timestamped HTML + JSON reports.

# Install git clone https://github.com/sanchaya/font-quality-inspector.git cd font-quality-inspector/font-checker && npm install # Analyze a font node index.js fonts/kan_gtn/fonts/ttf/KarnataGTN-Regular.ttf # Analyze a whole directory node index.js fonts/kan_gtn/fonts/ttf/ # Cross-font comparison: why does Font B render better? node index.js fontA.ttf --compare-font fontB.ttf # Version diff against a previous report node index.js font.ttf --compare reports/font_latest.json
